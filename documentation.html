How to use Scanner

If you are confused on what you see on the scanner, here are some helpful definitions to explain what everything is and what they do:

  1. SQL Attack: This type of attack is where an attacker inserts malicious SQL code into an input field, allowing them to manipulate the database and gain unauthorized access to sensitive information without authorization.
  2. XSS Attack: Also known as cross-site scripting, this is where malicious scripts are injected into trusted websites, causing them to execute the scripts in the victim's browser, potentially leading to unauthorized access to sensitive information.
  3. CSRF Attack: an attack that forces authenticated users to submit a malicious request to a Web application. In other words, it forces authenticated users to send dangerous requests to websites against the user's will.
  4. Level (1 - 5): This respresents how thorough the sql scan tests should be, the higher the level the more types of tests are performed.
  5. Risk (1 - 3): This represents how aggressive the scanner is, the higher the risk the more aggressive the scanner is.
  6. Forms: These allow users to enter data, which can then be sent to a server for processing; they are typically used to capture user input for various purposes, such as user registration, login, contact forms, survey forms and etc.
  7. Crawl: Also know as a spider, this process involved starting from a particular webpage, discovering new pages linked to it, adding those links to a queue, and repeating this process until all reachable pages have been crawled
